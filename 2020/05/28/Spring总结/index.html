1.2 基于注解配置

1.2.1 配置bean
  • @Component(value=”id”)

相当于xml给类配置一个bean,即将类添加到bean容器中

service层的类可用@Service(),controller层可用@Controller,持久层可用@repository，这三个注解与@Component作用一样。

1.2.2 成员变量值的注入

  • @Autowried

    自动注入类中全局变量。根据变量数据类型去IOC容器找，如果有唯一匹配的类，则注入该类映射的对象；如果有多个匹配的类，则根据变量名去匹配类的键名，有一致的匹配成功。否则注入失败。

  • @Qualifier(value= “bean ID”)

    给类的成员注入值时要结合@Autowried使用

    也可以加到方法参数上，将匹配的bean给参数赋值

  • @Resource(name =”bean ID”)

    可单独给类的成员注入值，但是注入的类型只能是配置好的bean，而不能是基本类型或string

    另外对于数据类型为集合类的成员，只能使用xml方式注入

  • @Value(value=””)

    用于给基本数据类型和String类型的成员变量赋值，支持EL表达式：${表达式}

1.2.3 bean的作用范围

@Scope(value=””)

在bean上加入，改变bean的作用范围

取值：singleton（单例,只创建一次对象，多次获取的是同一个对象），protoType（多例，每次获取时都会新创建一个对象）

1.2.4 生命周期相关

@PostConstruct

在方法上加入，类初始化时执行

@PreDesroy

类销毁时执行的对象，（比如 ClassPathXmlApplicationContext对象调用close方法关闭时）

1.2.5 其他注解

  • @Configuration

    在类上添加该注解，表明这是一个配置类，作用与bean.xml相同

    如果在创建容器时传入了该类的字节码

    则该类可以不加此注解。

  • @ComponentScan()
    指定创建容器时要扫描的包

    属性：basePackages/value={“包名”，“包名”，…}

    因为该注解只有一个属性，且为数组，故可以省略为：“xyz.zybolg”，“”

  • @Bean(name=”name”)

    加到方法上。作用，将该方法的返回值添加到Bean容器中，键为name；

    当方法有参数时，Spring会在Bean容器中查找参数对应的Bean，查找方式与Autowried一样。

  • @Import(“class1”,”class2”)

    属性：类字节码数组。作用，导入其他配置类（也可以将其他配置类的包添加到@ComponentScan中）

  • @PropertySource()

    属性：配置文件路径，如：“classpath:jdbcConfig.properties”（jdbcConfig.properties在resources下,程序运行后该目录下的资源会转移到类路径下，故前面要加上classpath:）

    配置好后，可以在EL表达式中取值，如：

对于以上两种配置方式的总结：纯注解配置反而增加了复杂度，最方便的配置方法是半注解半xml配置，即自定义的类用注解配置，jar包中的类用xml配置，使用bean.xml创建容器，不使用配置类

2. AOP

2.1 一些术语

2.1.1 Joinpoint(连接点)

所谓连接点是指那些被拦截到的点。在 spring 中,这些点指的是方法,因为 spring 只支持方法类型的 连接点。

2.1.2 Pointcut(切入点):

所谓切入点是指我们要对哪些 Joinpoint 进行拦截的定义。

2.1.3 Advice(通知/增强)

所谓通知是指拦截到 Joinpoint 之后所要做的事情就是通知。 通知的类型：前置通知,后置通知,异常通知,最终通知,环绕通知。

2.1.4 Introduction(引介)

引介是一种特殊的通知在不修改类代码的前提下, Introduction 可以在运行期为类动态地添加一些方 法或 Field。

2.1.5 Target(目标对象)

代理的目标对象。

2.1.6 Weaving(织入)

是指把增强应用到目标对象来创建新的代理对象的过程。 spring 采用动态代理织入，而 AspectJ 采用编译期织入和类装载期织入。

2.1.7 Proxy（代理）

一个类被 AOP 织入增强后，就产生一个结果代理类。

2.1.8 Aspect(切面)

是切入点和通知（引介）的结合。
